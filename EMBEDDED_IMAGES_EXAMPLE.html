Embedded vs Linked Images in HTML Reports

Explanation

This example demonstrates the difference between linked images and embedded images in HTML reports:

  • Linked Images: Reference external files using file paths
  • Embedded Images: Contain image data directly within the HTML file

The Issue You Experienced

When you shared your AutomationReport.html with your manager, they couldn't see the screenshots because:

  1. Your report used linked images that reference files on your computer
  2. These file paths don't exist on your manager's computer
  3. The browser shows broken image icons instead of the actual screenshots

Example of linked image (won't work on other computers):
<img src="../screenshots/example.png">

The Solution Implemented

I've updated the automation to include embedded images in addition to linked images:

  1. Screenshots are still saved to files locally
  2. Images are converted to Base64 format and embedded directly in the HTML
  3. Both versions are included in the report for maximum compatibility

Example of embedded image (works on any computer):
<img src="data:image/png;base64,iVBORw0KG...">

Now when you share the report, your manager will see the embedded images even without access to your file system.

Technical Details

The fix involved modifying the screenshot handling code to:

// Convert screenshot to Base64 for embedding
byte[] fileContent = Files.readAllBytes(Path.of(destPath));
String base64Image = java.util.Base64.getEncoder().encodeToString(fileContent);

// Add both linked and embedded versions to report
test.addScreenCaptureFromPath("../screenshots/" + fname, testName + " (Linked)");
test.addScreenCaptureFromBase64String(base64Image, testName + " (Embedded)");

Benefits of This Approach

  • Shareability: Reports work on any computer without external files
  • Self-contained: All information is in a single HTML file
  • Backwards compatibility: Still maintains linked images for local viewing
  • Professional presentation: Managers see complete reports without broken images
